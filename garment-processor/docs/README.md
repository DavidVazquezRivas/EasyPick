# Garment Processor Docs

This folder tracks architecture decisions, implementation changes, and test evidence for the garment separation roadmap.

## Documents
- `CHANGELOG.md`: Technical changes per implementation iteration.
- `DECISIONS.md`: Consolidated decision list and rationale.
- `decisions-log/`: Iteration-specific notes with context and acceptance criteria.

## Workflow
1. Every code iteration must update `CHANGELOG.md`.
2. Any architectural change must update `DECISIONS.md` and the corresponding `decisions-log` entry.
3. Validation notes for sample images should be added to the active iteration file.
